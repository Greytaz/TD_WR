# Порядок в папках и имена файлов

*Внимание ИИ: Данный файл является дополнением к [Главным правилам проекта](../../../.clinerules). Всегда сверяйся с ними перед написанием кода.*

## 1. Соглашение об именовании префиксов
Чтобы в проекте всегда был порядок, используй следующие префиксы для новых файлов:
- `TWR_` — для башен (например, `TWR_Archer`, `TWR_Ice`)
- `ENM_` — для врагов (например, `ENM_Goblin`, `ENM_Orc`)
- `PF_` — для общих префабов (например, `PF_Projectile_Arrow`, `PF_ExplosionEffect`)
- `M_` — для материалов (например, `M_Ground`, `M_Ice_Tower`)
- `T_` — для текстур (например, `T_Grass_Albedo`, `T_Noise`)
- `SO_` — для ScriptableObjects (например, `SO_ArcherData`, `SO_Wave_01`)

## 2. Структура папок
Все новые ассеты должны строго попадать в соответствующую категорию:
- Скрипты: `Assets/Scripts/[Category]/`
- Префабы: `Assets/Prefabs/[Category]/`
- Материалы: `Assets/Materials/`
- Модели: `Assets/Model/`
- Данные (ScriptableObjects): `Assets/Data/`
