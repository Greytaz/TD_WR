# Правила написания C# кода

*Внимание ИИ: Данный файл является дополнением к [Главным правилам проекта](../../../.clinerules). Всегда сверяйся с ними перед написанием кода.*

## 1. Понятный код с комментариями
- Пиши подробные комментарии на русском языке прямо в коде для каждой важной строчки.
- Объясняй сложные концепции простыми словами (в скобках) прямо в тексте чата и в комментариях к коду.
- Избегай сложных "фокусов" программирования (лямбды, сложные LINQ-запросы), если их можно написать простым и понятным циклом `foreach` (перебор элементов списка).

## 2. Стандарты именования (Code Style)
- Приватные переменные класса пиши с префиксом `m_` (например: `private float m_Health;`). Так пользователю будет легко отличить внутренние данные класса от временных переменных.
- Публичные переменные пиши с большой буквы (например: `public float Speed;`).
- Всегда оборачивай скрипты в пространства имен (namespace) `TD.Core`, `TD.Towers`, `TD.Enemies` в зависимости от папки.
- Используй автосвойства `public float Health { get; private set; }` для безопасного доступа к данным из других классов.
