# Архитектура Tower Defense

*Внимание ИИ: Данный файл является дополнением к [Главным правилам проекта](../../../.clinerules). Всегда сверяйся с ними перед написанием кода.*

## 1. Разделение данных и логики (ScriptableObjects)
- Все характеристики (здоровье врага, урон башни, стоимость) должны храниться в ScriptableObject (файлах-чертежах данных в папке `Assets/Data`).
- Сами скрипты на объектах (Monobehaviour) должны лишь считывать эти чертежи и хранить только текущее состояние в игре (например, сколько здоровья осталось прямо сейчас).

## 2. Связь через события (Events)
- Избегай жестких связей. Если башня убивает врага, она не должна напрямую лезть в GameManager.
- Вместо этого враг должен вызывать Событие (Event) "Я умер", а GameManager должен быть подписан на это событие, чтобы начислить игроку золото.
- Используй стандартные C# события (`public event Action OnDeath;`) для развязки систем.

## 3. Система Мета-прогресса (Skins, Fragments, Perks)
Мы заложили гибкую структуру для постоянной прокачки (Meta Progression):
- **Фрагменты (Fragments):** Накапливаются в `TowerMetaState` и расходуются в `TowerMetaManager` для повышения постоянного уровня башни за пределами боя.
- **Скины (SO_TowerSkin):** Содержат отдельные 3D-префабы моделей для каждого уровня башни (Tier 1, 2, 3). Игрок может разблокировать скины из сундуков и экипировать их.
- **Перки (SO_TowerPerk):** Пассивные карточки улучшений (например, урон по жукам-монстрам), которые можно надеть на башню.

## 4. Динамическая замена моделей в бою (Model Replacement - Type A)
- Чтобы поддерживать замену моделей при улучшении башни и использование разных скинов, мы НЕ уничтожаем саму башню `TowerBase` на карте (это сломало бы сетку).
- Вместо этого `TowerBase` при старте и прокачке вызывает `RebuildVisuals()`:
  1. Уничтожает старый дочерний объект модели.
  2. Запрашивает у `TowerMetaManager` нужную модель (с учетом текущего тира башни и выбранного скина игрока).
  3. Спавнит этот префаб как дочерний.
  4. Получает компонент `TowerVisuals` с новой модели, чтобы автоматически связать ключевые точки (`rotatableHead` для вращения к жукам, `firePoint` для вылета пуль и `animator` для анимаций выстрела и прокачки).
